## Key takeaways
1. Efficient: Solves problems in O(n) time by traversing the data once.
2. Space-saving: Requires O(1) space, often using just a few variables.
3. Real-time friendly: Ideal for streaming data or when only one pass is needed.
4. Minimizes overhead: Avoids redundant work by not revisiting data.
5. Optimal for aggregation: Useful for problems like finding sums, max/min, and counting elements.
6. Early termination: Stops processing once the solution is found (e.g., first match or condition met).
